Chapter 1: Getting Started

1.1 Welcome Aboard the Coding Train! 🚂

Hello, hello! Welcome to the wonderful world of programming! I'm so excited you're here! Are you ready to make beautiful, interactive, creative things with code? Of course you are! That's why you're here!

But wait, what even IS programming? Programming is writing instructions for a computer to follow. Think of it like writing a recipe, but instead of making cookies, you're making art, animations, games, and interactive experiences! The computer follows your instructions exactly - which is both amazing and sometimes frustrating when it does EXACTLY what you told it to do (even when that's not what you meant!).

What is p5.js?

p5.js is a JavaScript library that makes it super easy and fun to create graphics, animations, and interactive experiences. But what's a library? Think of a library like a toolbox - someone else has built a bunch of useful tools (functions) that you can use without having to build them yourself. Want to draw a circle? There's a tool for that! Want to make something move? There's a tool for that too!

p5.js was created by Lauren McCarthy and is based on Processing, a programming language designed by Casey Reas and Ben Fry. The goal? To make coding accessible to artists, designers, educators, and beginners. It's all about making coding visual and immediate - you write code and instantly see the results on your screen!

1.2 The p5.js Web Editor - Your New Best Friend

First things first! Let's get you set up with the p5.js Web Editor. This is where the magic happens!

Getting Started with the Editor

  1. Open your web browser (Chrome, Firefox, Safari, Edge - they all work!)
  2. Go to https://editor.p5js.org/
  3. That's it! No downloads, no installations, no "configuring your development environment" nonsense. Just pure, unadulterated creative coding joy!

Understanding the Interface

When you first open the editor, you'll see several parts:

The Code Area (Left Side): This is where you write your code. You'll see some starter code that looks like this:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
}

The Preview Area (Right Side): This shows what your code creates. Right now, it's just a gray square, but soon it'll be full of your amazing creations!

The Console (Bottom): This is where error messages appear. Think of it as the place where the computer talks back to you when something goes wrong.

The Toolbar (Top):

  • Play Button (▶️): Runs your code
  • Stop Button (⏹️): Stops your code
  • Auto-refresh checkbox: When checked, your sketch updates automatically as you type

Your First Sketch

Let's understand that starter code:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
}

This is the basic structure of EVERY p5.js sketch. Let's break it down:

function setup(): This function runs ONCE when your program starts. It's where you set things up (hence the name!). Right now, it's creating a canvas.

createCanvas(400, 400): This creates a drawing area that's 400 pixels wide and 400 pixels tall. Think of pixels as tiny dots on your screen - the more pixels, the bigger your canvas!

function draw(): This function runs OVER AND OVER, approximately 60 times per second! It's constantly drawing to your canvas.

background(220): This fills the entire canvas with a color. The number 220 represents a shade of gray (we'll learn more about colors soon!).

Hit the play button (▶️) and... LOOK! You made a gray square! I know, I know, not the most exciting thing in the world, but hey, it's a start!

1.3 Let's Draw Some Shapes!

Gray rectangles are cool and all, but let's make something more exciting! Let's draw some shapes!

Drawing a Circle

Delete everything in the draw function and add this:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(200, 200, 100);
}

Boom! A circle! But what do those numbers mean?

The circle() function needs three pieces of information:

  1. x position: How far from the left edge (200 pixels)
  2. y position: How far from the top edge (200 pixels)
  3. diameter: How wide the circle is (100 pixels)

So circle(200, 200, 100) means: "Draw a circle at position (200, 200) with a diameter of 100 pixels."

Try it! Change the numbers! What happens if you change the first 200 to 100? What about making the diameter 200?

Understanding the Coordinate System

Wait, wait, wait. "From the left?" "From the top?" Let me explain how the coordinate system works in p5.js!

In p5.js (and most computer graphics), the coordinate system starts at the top-left corner:

(0,0) ---------- (400,0) | | | | | (200,200) | | | | | (0,400) -------- (400,400)
  • The origin (0,0) is at the TOP-LEFT corner
  • X increases as you go RIGHT
  • Y increases as you go DOWN (this might feel backwards if you're used to math class!)

Let's visualize this:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Top-left corner
  circle(0, 0, 50);
  
  // Top-right corner
  circle(400, 0, 50);
  
  // Bottom-left corner
  circle(0, 400, 50);
  
  // Bottom-right corner
  circle(400, 400, 50);
  
  // Center
  circle(200, 200, 50);
}
Notice: The circles at the corners are partially cut off! That's because the x and y positions refer to the CENTER of the circle!

Drawing Rectangles

Let's draw a rectangle:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  rect(50, 50, 150, 100);
}

The rect() function needs four pieces of information:

  1. x position: X coordinate of the top-left corner (50 pixels from left)
  2. y position: Y coordinate of the top-left corner (50 pixels from top)
  3. width: How wide the rectangle is (150 pixels)
  4. height: How tall the rectangle is (100 pixels)
Important difference: Rectangles are positioned by their TOP-LEFT corner, while circles are positioned by their CENTER!

Drawing Lines

Lines connect two points:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  line(50, 50, 350, 350);
}

The line() function needs four pieces of information:

  1. x1: X coordinate of the first point (50)
  2. y1: Y coordinate of the first point (50)
  3. x2: X coordinate of the second point (350)
  4. y2: Y coordinate of the second point (350)

This draws a diagonal line from the top-left towards the bottom-right!

Drawing Triangles

Triangles are defined by three points:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  triangle(200, 50, 100, 300, 300, 300);
}

The triangle() function needs six pieces of information - the x and y coordinates of three points:

  1. First point: (200, 50) - top of the triangle
  2. Second point: (100, 300) - bottom-left
  3. Third point: (300, 300) - bottom-right

More Shapes

Let's put multiple shapes together:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Rectangle
  rect(50, 50, 100, 75);
  
  // Circle
  circle(250, 100, 80);
  
  // Line
  line(50, 200, 150, 250);
  
  // Triangle
  triangle(250, 200, 200, 280, 300, 280);
}

Order Matters!

Important concept: Shapes are drawn in the order they appear in your code. Later shapes are drawn ON TOP of earlier shapes! 
function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Rectangle drawn first
  rect(150, 150, 100, 100);
  
  // Circle drawn second (on top)
  circle(200, 200, 100);
}

The circle appears on top of the rectangle because it's drawn after! Try swapping the order of the rect() and circle() lines. See what happens!

1.4 Adding Color - Let's Make It Pop!

Gray is nice, but COLOR is where the party's at! Let's learn how to add color to our sketches.

Fill - Coloring the Inside

The fill() function sets the color inside shapes:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  fill(255, 0, 0);
  circle(200, 200, 100);
}

A red circle! But what do those three numbers mean?

Understanding RGB Colors

Colors in p5.js use the RGB color model:

  • Red: How much red (0-255)
  • Green: How much green (0-255)
  • Blue: How much blue (0-255)

Think of it like mixing colored lights:

  • fill(255, 0, 0) = Maximum red, no green, no blue = Pure red!
  • fill(0, 255, 0) = No red, maximum green, no blue = Pure green!
  • fill(0, 0, 255) = No red, no green, maximum blue = Pure blue!

What about other colors?

  • fill(255, 255, 0) = Red + Green = Yellow!
  • fill(255, 0, 255) = Red + Blue = Magenta!
  • fill(0, 255, 255) = Green + Blue = Cyan!
  • fill(255, 255, 255) = All colors max = White!
  • fill(0, 0, 0) = No colors = Black!

Let's try it:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Red circle
  fill(255, 0, 0);
  circle(100, 100, 80);
  
  // Green circle
  fill(0, 255, 0);
  circle(200, 100, 80);
  
  // Blue circle
  fill(0, 0, 255);
  circle(300, 100, 80);
  
  // Yellow rectangle
  fill(255, 255, 0);
  rect(75, 200, 50, 50);
  
  // Magenta rectangle
  fill(255, 0, 255);
  rect(175, 200, 50, 50);
  
  // Cyan rectangle
  fill(0, 255, 255);
  rect(275, 200, 50, 50);
}

Grayscale Colors

If you use just one number instead of three, you get shades of gray:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  fill(0);      // Black
  circle(100, 200, 80);
  
  fill(127);    // Medium gray
  circle(200, 200, 80);
  
  fill(255);    // White
  circle(300, 200, 80);
}

Stroke - Coloring the Outline

The stroke() function sets the outline color:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  fill(255, 0, 0);      // Red inside
  stroke(0, 0, 255);    // Blue outline
  circle(200, 200, 100);
}

Stroke Weight - Thickness of Lines

You can make outlines thicker or thinner:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  fill(255, 0, 0);
  stroke(0, 0, 255);
  strokeWeight(5);      // Thick outline!
  circle(200, 200, 100);
}

No Fill and No Stroke

Sometimes you want just an outline or just a fill:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Just outline, no fill
  noFill();
  stroke(0);
  strokeWeight(3);
  circle(150, 200, 80);
  
  // Just fill, no outline
  noStroke();
  fill(255, 0, 0);
  circle(250, 200, 80);
}

Color Affects All Shapes After It

Important: When you set a color, it applies to ALL shapes drawn after it until you change it again! 
function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  fill(255, 0, 0);  // Set fill to red
  circle(100, 100, 50);
  circle(200, 100, 50);  // Also red!
  
  fill(0, 255, 0);  // Change fill to green
  circle(300, 100, 50);  // Now it's green
  
  rect(100, 200, 50, 50);  // This is also green!
}

Putting It All Together

Let's make a colorful drawing:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(135, 206, 235);  // Sky blue
  
  // Sun
  fill(255, 223, 0);
  noStroke();
  circle(350, 50, 60);
  
  // House body
  fill(178, 34, 34);
  rect(150, 200, 100, 100);
  
  // Roof
  fill(101, 67, 33);
  triangle(150, 200, 250, 200, 200, 150);
  
  // Door
  fill(101, 67, 33);
  rect(190, 250, 20, 50);
  
  // Windows
  fill(135, 206, 250);
  stroke(0);
  strokeWeight(2);
  rect(165, 220, 25, 25);
  rect(210, 220, 25, 25);
  
  // Grass
  fill(34, 139, 34);
  noStroke();
  rect(0, 300, 400, 100);
}

Look at that! You just drew a colorful house scene with code!

1.5 Errors and the Console - Don't Panic!

Let's be real for a second. You're going to make mistakes. I make mistakes. Everyone makes mistakes! And that's TOTALLY OKAY! In fact, errors are your friends because they help you learn!

Common Errors

Let's intentionally make some mistakes to see what happens:

Typo in Function Name

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  cirle(200, 200, 100);  // Oops! Misspelled 'circle'
}

Error message: cirle is not defined

The computer doesn't know what cirle is because the function is called circle!

Missing Comma

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(200 200, 100);  // Oops! Forgot comma between 200 and 200
}

Error message: Unexpected number

The computer is confused because it expected a comma but found another number!

Missing Parenthesis

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(200, 200, 100;  // Oops! Forgot closing parenthesis
}

Error message: Unexpected token ';'

The computer expected a closing parenthesis but found a semicolon instead!

Reading Error Messages

p5.js has friendly error messages! They usually tell you:

  • What went wrong
  • Which line has the problem
  • Sometimes even suggestions for fixing it!

For example:

🌸 p5.js says: It looks like there might be a typo in your code. "cirle" is not defined. Did you mean "circle"?

How helpful is that?!

The Console

The console at the bottom of the editor shows:

  • Error messages (in red)
  • Any messages from your program (we'll learn about this later)
  • Helpful hints from p5.js
